# VERCEL_ONLY_MIGRATION.md

## ملخص

تم التأكد أن المشروع يعمل على Vercel فقط ولا يعتمد على Cloudflare نهائيًا.

- لا توجد أي ملفات أو إعدادات أو متغيرات بيئية مرتبطة بـ Cloudflare.
- جميع عمليات البناء والنشر تتم عبر Vercel فقط.
- جميع السكريبتات والـ API routes متوافقة مع بيئة Vercel.

## إعدادات DNS المطلوبة

### إذا كان DNS مُدارًا عبر Vercel (nameservers عند Vercel):
- أضف الدومين الرئيسي (root) والدومين الفرعي www من لوحة تحكم Vercel.
- Vercel سيعطيك nameservers (عادةً ns1.vercel-dns.com ... إلخ)، قم بتحديثها في مزود الدومين.
- لا حاجة لأي سجلات A أو CNAME يدوية.

### إذا كان DNS خارجي (nameservers خارج Vercel):
- أضف الدومين في Vercel وحدد أنه DNS خارجي.
- أضف السجلات التالية في مزود الدومين:
  - للـ root (@):
    - CNAME → cname.vercel-dns.com
    - أو A → عناوين IP التي توفرها Vercel (يفضل CNAME إذا كان متاحًا)
  - للـ www:
    - CNAME → cname.vercel-dns.com
- تحقق من حالة الدومين من لوحة تحكم Vercel حتى تظهر علامة ✅.

## خطوات يدوية لازمة منك

- إذا لم يكن الدومين مُدارًا بالكامل عبر Vercel، يجب عليك تحديث إعدادات DNS يدويًا كما هو موضح أعلاه.
- بعد تحديث DNS، انتظر حتى يتم تفعيل SSL تلقائيًا من Vercel.
- تحقق من عمل الموقع عبر الدومين root و www.

## ملاحظات
- لا توجد أي بقايا أو إعدادات نشر قديمة لـ Cloudflare.
- إذا واجهت أي مشكلة في ربط الدومين، راجع لوحة تحكم Vercel أو تواصل مع الدعم.
